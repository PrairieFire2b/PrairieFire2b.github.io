# 编程语言 #1 设计

        在编写语言cpile的过程中，因为种种原因，直到现在，我还没有设计完语法。因为我认为，设计一门编程语言应当考虑三要素：
    简明，回溯，语义。
        简明性，即为简洁明了。作为设计者，我们应尽力把语法合理简化，且让人能够快速读懂代码。同时我们也应该提供一些语法糖，
    这可以让使用者和语言都减少些工作负担。例如设计匿名函数的语法，如果在下面的语法设计中选择一种，我会更倾向于JS的语法。
```cpp
const auto lambda = [?, this]() {}; // cpp
```
```javascript
const lambda = (() => {}).bind(this); // js
```
        显而易见，JS的代码只需稍加解释bind的作用，你就能读懂。而CPP的代码还需要你去记?的作用，this的作用，mutable的作用
    等等，甚至还有其它的符号作用。所以从中我们可以发现，让代码简洁明了，不将功能耦合在一起的重要性与优点。
        回溯，即为追溯代码。作为使用者，我们会希望让错误易于追溯与检查。但有的时候，问题可能不是使用者编写的代码逻辑问题，
    而是使用的语法，用了相同的符号，导致优先级冲突。例如设计指针定义的语法，如果是使用下面的语法，就可能会产生优先级问题。
```cpp
int ptr*=0;
```
        这种情况下，如果语言支持*=运算符，且指针被设计为整型，你的代码就可能产生问题。作为设计者，我们应尽量避免和减少这样
    的问题，且添加更多错误提示。但很少有人设计出示例这样的语法，因为这样的语法冲突问题通常会很明显，我举的示例也并不生动，
    现实中问题往往还要比这复杂的多。所以从中我们就可以了解，让符号使用唯一化，增强错误回溯的必要性与好处。
        语义性，即为语法意义。现在，很多语言使用作为关键字的词汇，都大同小异，通常只是语法稍有不同，但是它们可能都为同一个
    目的服务，这个目的就是语义的背景，也就是望文生义。例如下面给出的代码，它们都来自于不同的语言，你可以猜猜有什么作用。
```javascript
var inst = new String(); // js
```
```rust
let mut inst = String::new(); // rs
```
        尽管写法五花八门，我也没解释这两行代码，但是即使没有学过JS，RS，你也能一眼判断出来这行代码的作用————即实例化一个
    字符串类或者结构体的对象。new决定了这两行代码的背景语义，你负责规定真正具体的语义。所以总结就是，在使用这些非常常用的
    词汇作为语言的关键字时，切记不要与其它语言里的用途太大相径庭，这可能会对没有阅读过官方教程或文档的使用者造成理解障碍。
        同时，语法的改变也同样会改变它的背景语义，影响到人们对它的部分理解。例如我们想要一个让变量被初始化为指针，如下文。
```typescript
const ptr = new pointer<void>(0);
```
```cpp
const auto ptr = (void*)(0);
```
        如你所见，前一行代码显然是构造了泛型指针类的实例并赋值，且类型为void。后一行的代码却更像是使用特殊语法，把变量赋值
    为一个int类型的指针。即使它们起到的作用相同，但使用的语法不同，理解的语义也不尽相同。如果你没有学过cpp，那通常你会觉得
    第一行的代码更易于理解。所以在设计语法时，应尽可能地明确语义，并使语法贴合日常使用习惯，这让人能够快速适应我们的语言。
        那么，我们还可以从以上三点中得出：一个关键字，通常应只运用于一种语法，除非你打算设计该语法的扩展。例如导入关键字。
```javascript
import lib
```
```javascript
var name = import lib
```
        后一行代码显然就是前一行代码的语法使用延展。但如果你觉得这个示例还不能完全展示出，完全延展的意义，那么请看下一个。
```js
import lib as name
```
```js
export lib as name
```
        你可以发现，在这种情况下，as的语法也相当于被延展了，它被运用到别的语法上，而不改变其现有语法。这让语法更易于记忆。
    同时，如果你使用了一个有与其它词汇重合部分的缩写作为关键字，即使两个关键字不同，也有可能造成理解障碍。例如导入关键字。
```
imp name for sth
```
        你能从这个语句中，判断出它到底是为sth实现name，还是在外部导入name到sth里吗？显然不能，因为import和impl有重合部分，
    且两个解释都可能存在，这让我们难以定论。所以，尽量不要使用太少字数容易意味不明，或与其它关键字有重合的缩写作为关键字。
    而在这部分里，for也不是前面提到的语法延展。它本身是循环的关键字，在这里却被用作新的语法，这也同样是我们需要避免的情况。
